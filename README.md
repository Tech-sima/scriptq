# Bot Checker — Черно-голубой (Static + Firebase + Vercel endpoint)

Коротко: простая страница с двумя колонками. Левая — ввод списка ботов (по одному в строке). Правая — отображение ботов и их статуса: `свободен` или `занят`. Незаполненные (без владельца) боты при проверке автоматически закрепляются за текущим пользователем. Данные синхронизируются в реальном времени через Firebase Realtime Database.

Файлы:
- `index.html` — интерфейс
- `styles.css` — стиль (черно-голубая тема)
- `app.js` — логика фронтенда и синхронизация с Firebase
- `api/uuid.js` — простой серверный endpoint (Vercel)

Установка и запуск локально
1. Скопируйте папку в рабочую директорию (она у вас уже в `scriptq`).
2. Создайте Firebase проект (https://console.firebase.google.com).
3. Включите Realtime Database и в тестовом режиме укажите правила (для простоты тестирования):

```
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

4. Скопируйте конфиг Firebase (Settings -> General -> Your apps) и вставьте значения в `firebaseConfig` в `app.js`.
   - Обязательно укажите `databaseURL` вида `https://YOUR_PROJECT_ID-default-rtdb.firebaseio.com`.

5. Откройте `index.html` в браузере (можно локально) — интерфейс будет подключаться к Firebase и синхронизироваться.

Как это работает
- При загрузке списка (левая колонка) создаются записи `/bots/{encodedName}` с полем `name`.
- При нажатии `Проверить` выполняется транзакция по каждому боту: если `ownerId` пуст, она устанавливается в `uid` текущего анонимного пользователя.
- Клиенты подписаны на `/bots` и получают обновления в реальном времени.

Деплой на Vercel
1. Зарегистрируйтесь в Vercel и создайте новый проект, подключив этот репозиторий/папку.
2. Vercel автоматически развернёт статические файлы и endpoints (файлы в `api/` создают serverless-функции).
3. Перед деплоем замените `firebaseConfig` в `app.js` на ваш реальный конфиг (или используйте env vars и включите их в билде).

Примечания и безопасность
- В этом демо правила Realtime Database предложены открытыми для упрощения. Для реального проекта настройте правила доступа и используйте Firebase Authentication более явно.
- `api/uuid.js` — просто демонстрационный backend-файл, который необходим для примера serverless-эндпоинта на Vercel.
